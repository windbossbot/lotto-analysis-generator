# Preview Workflow

화면 수정 작업은 아래 순서로 진행합니다.

## 기본 원칙

- 사용자가 "보여줘", "켜줘", "미리보기"를 요청하면 먼저 로컬 웹을 연다.
- 미리보기 서버는 사용자가 확인을 마칠 때까지 임의로 끄지 않는다.
- 수정 중에는 서버를 계속 유지하고, 사용자가 "꺼", "이제 종료"라고 할 때만 종료한다.
- 링크가 아니라 확인 가능한 결과가 필요할 때는 브라우저를 직접 열거나 스크린샷을 만든다.

## 작업 순서

1. 서버 상태 확인
2. 서버가 꺼져 있으면 다시 실행
3. 브라우저 열기
4. 화면 수정
5. 사용자가 새로고침해서 바로 볼 수 있게 유지
6. 필요하면 스크린샷 추가 제공
7. 사용자가 종료를 요청할 때만 `npm run stop:port` 실행

## 로컬 명령

```bash
npm run stop:port
npm run start:safe
npm run preview
```

`npm run preview`는 아래를 한 번에 처리합니다.

- 서버 헬스체크
- 서버가 내려가 있으면 포트 정리 후 재시작
- 준비될 때까지 대기
- 기본 브라우저에서 미리보기 열기

직접 서버를 켜야 할 때:

```bash
node server.js
```

종료:

```bash
npm run stop:port
```

## 주의사항

- 작업 완료 직후 자동 종료하지 않는다.
- 브라우저가 열려 있더라도 서버가 내려가면 사용자 입장에서는 "안 보임" 상태가 된다.
- 수정 후 확인 단계에서는 "서버 유지"를 기본값으로 둔다.
